## Codebase Patterns
- Use `sql<number>` template for aggregations
- Always use `IF NOT EXISTS` for migrations
- Export types from actions.ts for UI components
- Use @patch.object() to mock methods on Dagster ConfigurableResource subclasses
- Use real resource objects with mocked methods (not MagicMock with spec) for Dagster asset tests
- Asset dependencies: deps=[AssetKey("upstream_asset")] in the @dg.asset decorator
- MetadataValue.json() for structured data in MaterializeResult
- Schedules registered via Definitions(schedules=[...])
- Jobs registered via Definitions(jobs=[...])
- Type hint dict[str, Any] for flexible dictionary values in asset metadata
- define_asset_job() returns UnresolvedAssetJobDefinition - assert on name, not isinstance JobDefinition
- AssetSelection.groups('streamify').downstream() for selecting group assets with dependencies
